# OpsMind 模块实施计划

本文档按照可交付模块拆分开发任务。建议严格按顺序推进：先完成基础工程和数据层，再实现知识库、检索和 AI 能力。

职责划分：

- **你负责**：后端、数据库、基础设施、部署和 Git 操作。
- **AI 助手负责**：前端工程、页面、组件、样式、接口适配和前端验证。

## M0：项目骨架

### 目标

建立可启动、可测试、可持续扩展的前后端工程。

### 需要完成

你需要完成：

- 创建 `backend/`，初始化 FastAPI 项目。
- 创建 `deploy/`，编写本地开发使用的 Docker Compose。
- 在 Compose 中加入 PostgreSQL、Redis 和 ChromaDB。
- 添加 `.env.example`，列出必要环境变量，不填写真实密钥。
- 为后端添加健康检查接口，例如 `GET /api/v1/health`。

AI 助手完成：

- 创建 `frontend/`，初始化 Vue 3 项目。
- 实现基础布局和后端健康状态展示。

### 完成标准

- 前端可以打开默认页面。
- 后端健康检查返回 `200`。
- PostgreSQL、Redis 和 ChromaDB 可以在本地启动。

## M1：用户与权限

### 目标

让系统具备基本登录能力，并限制管理操作。

### 需要完成

- 设计用户、角色和权限数据表。
- 实现用户登录和身份校验。
- 区分普通用户与管理员。
- 为 Wiki 写操作和系统配置接口添加权限检查。
- 添加基础审计日志。

### 完成标准

- 未登录用户无法访问受保护接口。
- 普通用户无法执行管理员操作。
- 登录、创建文档和删除文档会记录审计日志。

### 前端配合

后端接口稳定后，AI 助手实现登录页、身份状态和基础路由守卫。

## M2：Wiki 与文档管理

### 目标

实现知识库的基础增删改查。

### 需要完成

- 设计 Wiki 页面、分类、标签、附件和版本记录数据表。
- 实现页面创建、编辑、查询、删除和版本查看接口。
- 支持上传 Markdown、PDF、DOCX 和 TXT 文件。
- 校验文件类型和大小。
AI 助手根据后端接口实现页面列表、编辑器、详情页和上传入口。

### 完成标准

- 用户可以创建和编辑 Wiki 页面。
- 用户可以上传受支持文件。
- 系统能够展示页面版本记录。
- 非法文件类型会被拒绝。

## M3：文档解析与索引

### 目标

将上传文档转化为可检索的文本片段和向量数据。

### 需要完成

- 为不同文件格式实现解析器。
- 设计文本切分策略，保留文档 ID、标题和片段位置。
- 接入 Embedding 模型。
- 将向量及元数据写入 ChromaDB。
- 为异步索引任务增加状态：`pending`、`processing`、`ready`、`failed`。
AI 助手根据后端接口展示索引状态。

### 完成标准

- 上传文档后可以异步完成索引。
- 索引失败时可以看到错误状态。
- 每个向量片段都能追溯到原始文档和位置。

## M4：知识检索

### 目标

实现关键词检索、语义检索和结果融合。

### 需要完成

- 实现 Wiki 关键词检索。
- 实现 ChromaDB 语义检索。
- 设计混合检索排序策略。
- 返回文档名称、片段位置、摘要和相似度。
AI 助手根据后端接口实现搜索页面。

### 完成标准

- 输入典型运维问题可以返回相关文档。
- 每条搜索结果都包含可定位的引用信息。
- 可以分别测试关键词检索和语义检索。

## M5：AI 问答

### 目标

基于知识库生成带引用来源的回答。

### 需要完成

- 创建统一的 LLM Provider 适配层。
- 从检索结果构建问答上下文。
- 要求模型在知识不足时明确说明不确定性。
- 将回答和引用来源一并返回。
- 记录模型、耗时、错误和引用信息。
AI 助手根据后端接口实现问答页面。

### 完成标准

- 回答会展示引用文档和引用位置。
- 没有足够知识时不会编造确定答案。
- 更换 Provider 时不需要修改业务逻辑。

## M6：故障案例库

### 目标

沉淀可复用的历史故障处理经验。

### 需要完成

- 设计故障现象、原因、排查过程、修复方案和复盘结论字段。
- 实现案例增删改查。
- 将案例纳入索引和检索范围。
- 根据当前问题推荐相似案例。

### 完成标准

- 用户可以完整记录一条故障案例。
- AI 问答可以引用历史故障案例。
- 系统可以返回相似案例列表。

### 前端配合

后端接口稳定后，AI 助手实现案例列表、详情和编辑页面。

## M7：Agent 工具调用

### 目标

让 Agent 根据问题调用只读知识工具。

### 需要完成

- 实现 Wiki Search Tool。
- 实现 Document Search Tool。
- 实现 Knowledge Summary Tool。
- 记录工具名称、参数、结果摘要和耗时。
- 限制首期工具为只读操作。

### 完成标准

- Agent 可以根据问题选择合适工具。
- 每次工具调用都可追踪。
- Agent 无法执行系统命令或修改生产环境。

## M8：部署与可观测性

### 目标

让系统具备可部署和可排障能力。

### 需要完成

- 完善 Dockerfile 和 Docker Compose。
- 使用 Nginx 代理前端和后端。
- 增加结构化日志。
- 接入 Prometheus 指标和 Grafana 仪表盘。
- 编写备份、恢复和发布说明。

### 完成标准

- 新环境可以按照文档完成部署。
- 可以查看 API、索引任务和模型调用的关键指标。
- PostgreSQL 数据可以备份和恢复。

### 前端配合

AI 助手负责前端生产构建和 Nginx 静态资源相关配置示例。

## 后续扩展

完成 M0 至 M8 后，再评估以下能力：

- MCP 接入
- 多 Agent 协同
- Kubernetes 运维助手
- 企业知识图谱
- 受控 SSH 远程执行
- 带人工审批的自动故障处理
